# Vision

`scrapling-kotlin` 的愿景是：**让 Kotlin 社区获得与 `Scrapling` 源项目同等级、同语义、同使用路径的抓取能力，而不是一个阉割版移植品。**

当愿景达成时，以下陈述必须同时成立：

1. Kotlin 开发者可以完成源项目文档覆盖的主要工作流：解析、静态抓取、动态抓取、stealth 抓取、爬虫调度、CLI 提取、MCP 服务。
2. 对应能力都存在自动化测试与夹具证据，且测试组织结构能一一映射源项目的能力域。
3. Kotlin 版本对外呈现的 API 和行为语义与源项目一致；如因语言差异必须调整调用方式，必须提供兼容适配层或迁移说明。
4. 文档、计划、测试、代码之间始终可追溯，不允许“代码有了，文档没跟上”或“文档写了，代码跑偏”。

非愿景：

- 不接受只做解析器子集就宣称“已有 Scrapling Kotlin 版”。
- 不接受把浏览器抓取、Spider、CLI、MCP 永久放进“以后再说”。
- 不接受以 Kotlin idiomatic 为借口删除源项目的核心能力。
